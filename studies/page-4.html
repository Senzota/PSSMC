Chapters 8 – 9  ·  PSSMC Mathematics

Lyapunov Stability
& System Parameters

A simulation shows a controller works. A Lyapunov proof shows it must always work — for any initial condition, any bounded disturbance, forever. This chapter builds that guarantee from scratch, step by step, with plain-language explanation alongside every equation.

8.1  Why a Lyapunov Proof?

A simulation demonstrates controller performance at specific initial conditions and a specific disturbance profile. It cannot guarantee stability universally — a different starting point, a different chatter force, or a numerical accident could in principle produce a diverging trajectory.

A Lyapunov proof provides a mathematical guarantee: for any initial condition and any bounded disturbance, the system converges to a neighbourhood of the sliding surface and stays there. This transforms an empirically good controller into a theoretically certified one.

⛰️

The Mountain Valley Analogy

Imagine a ball on a landscape. If you can show there is a "valley" — a bowl-shaped energy function \(V\) with the equilibrium at the bottom — and that the ball always rolls downhill (\(\dot{V} \leq 0\)), then no matter where you release the ball, it must eventually reach the valley floor. That is exactly what the Lyapunov proof does for the PSSMC. We define \(V(t)\), show it is a bowl, and show the system always rolls toward \(s = 0\).

8.2  The Lyapunov Candidate Function — Eq. (33–34)

The key challenge in Lyapunov analysis is choosing the right \(V(t)\). For a standard SMC, \(V = \tfrac{1}{2}s^2\) suffices. But the PSSMC has three additional adaptive parameters — \(\lambda\), \(T_p\), \(\phi\) — each of which deviates from its ideal value. The candidate function must account for all of these simultaneously:

Lyapunov candidate — Equation (33) \[ V(t) = \frac{1}{2}s^2(t) + \frac{1}{2\gamma_1}\tilde{\lambda}^2(t) + \frac{1}{2\gamma_2}\tilde{T}_p^2(t) + \frac{1}{2\gamma_3}\tilde{\phi}^2(t) \]
Parameter errors — Equation (34) \[ \tilde{\lambda} = \lambda - \lambda^*, \qquad \tilde{T}_p = T_p - T_p^*, \qquad \tilde{\phi} = \phi - \phi^* \]

where \(\lambda^*\), \(T_p^*\), \(\phi^*\) are the ideal parameter values for the current operating condition — the values a perfectly-informed designer would choose. The \(\gamma_i > 0\) are scalar weights that control how much each parameter error contributes to the total "energy" of the system.

Anatomy of \(V(t)\) — Four Terms, Four Roles

\(\tfrac{1}{2}s^2\)

Surface energy. The core term — measures how far the system is from the sliding surface. When \(s = 0\), this term is zero. The larger the chatter (bigger \(|s|\)), the larger this term.

\(\tfrac{1}{2\gamma_1}\tilde{\lambda}^2\)

Sliding gain error. Measures how far \(\lambda\) has drifted from its ideal value \(\lambda^*\). Prevents \(\lambda\) from adapting to an arbitrarily wrong value. Always non-negative.

\(\tfrac{1}{2\gamma_2}\tilde{T}_p^2\)

Horizon error. Measures the prediction horizon error. Ensures \(T_p\) is pulled toward the ideal prediction time rather than collapsing to zero or growing without bound.

\(\tfrac{1}{2\gamma_3}\tilde{\phi}^2\)

Boundary layer error. Measures how far \(\phi\) is from its ideal value. Prevents the boundary layer from either shrinking to zero (causing chattering) or inflating indefinitely (losing precision).

Two Lyapunov conditions satisfied immediately:
(1) \(V(t) \geq 0\) always — \(V\) is a sum of squares scaled by positive constants. A sum of squares is never negative. ✓
(2) \(V(t) = 0\) if and only if \(s = 0\) and \(\tilde{\lambda} = \tilde{T}_p = \tilde{\phi} = 0\) — i.e., the system is perfectly on the surface with all parameters at their ideal values. ✓
The hard part is proving the third condition: \(\dot{V}(t) \leq 0\) outside a bounded region. That is what Sections 8.3 and 8.4 establish.

8.3  Time Derivative of \(V(t)\) — Eq. (35–37)

To prove stability, we differentiate \(V(t)\) and show it is negative — meaning the total "energy" of the system is always decreasing. Think of it as showing the ball is always rolling downhill.

1
Differentiate Eq. (33) directly

Each squared term differentiates to a product: \(\frac{d}{dt}\frac{1}{2}s^2 = s\dot{s}\), and similarly for the parameter errors.

Eq. (35) \[ \dot{V}(t) = s\dot{s} + \frac{1}{\gamma_1}\tilde{\lambda}\dot{\tilde{\lambda}} + \frac{1}{\gamma_2}\tilde{T}_p\dot{\tilde{T}}_p + \frac{1}{\gamma_3}\tilde{\phi}\dot{\tilde{\phi}} \]
Note: \(\dot{\tilde{\lambda}} = \dot{\lambda}\) since \(\lambda^*\) is a constant ideal value. Same for \(T_p^*\) and \(\phi^*\).
2
Substitute the adaptive laws (28–30) from Page 3

Replace \(\dot{\lambda} = -\alpha_1 s^2\), \(\dot{T}_p = -\alpha_2 s^2\), \(\dot{\phi} = -\alpha_3|s|\):

Eq. (36) \[ \dot{V}(t) = s\dot{s} + \frac{1}{\gamma_1}\tilde{\lambda}(-\alpha_1 s^2) + \frac{1}{\gamma_2}\tilde{T}_p(-\alpha_2 s^2) + \frac{1}{\gamma_3}\tilde{\phi}(-\alpha_3|s|) \]
3
Set \(\gamma_i = \alpha_i\) — the gains cancel cleanly

This is the standard choice in adaptive SMC literature. It causes the weight denominators to cancel with the adaptation gains, leaving a clean expression:

Eq. (37) \[ \dot{V}(t) = s\dot{s} - \tilde{\lambda}\,s^2 - \tilde{T}_p\,s^2 - \tilde{\phi}\,|s| \]
The last three terms are always dissipative when the parameter errors have the same sign as the surface — this is what the lower bounds on \(\lambda\), \(T_p\), \(\phi\) guarantee.

8.4  Bounding \(\dot{V}(t)\) — Eq. (38–40)

The \(s\dot{s}\) term is what connects the Lyapunov function to the actual closed-loop dynamics. We need to substitute the closed-loop expression for \(\dot{s}\) and then use a key inequality to show the result is negative.

4
Closed-loop surface dynamics — what does \(\dot{s}\) look like under PSSMC?

Let \(d(t)\) represent all bounded disturbances and modelling errors, satisfying \(|d(t)| \leq \Delta\) for some known bound \(\Delta\). Under the PSSMC control law, the surface dynamics become:

Eq. (38) \[ \dot{s}(t) = d(t) - K\,\tanh\!\left(\frac{s(t)}{\phi}\right) \]
Intuitively: the disturbance \(d(t)\) tries to push \(s\) away from zero; the switching term \(K\,\tanh(s/\phi)\) opposes that push. Whether \(s\) grows or shrinks depends on which term wins.
5
The key tanh inequality

For any real \(s\) and any \(\phi > 0\), the following always holds:

Eq. (39) \[ s\,\tanh\!\left(\frac{s}{\phi}\right) \geq 0, \quad \text{with equality only at } s = 0 \]
Why? Because \(s\) and \(\tanh(s/\phi)\) always have the same sign: both positive when \(s > 0\), both negative when \(s < 0\), both zero at \(s = 0\). Their product is therefore always non-negative. This simple inequality is the keystone of the entire stability proof.
6
Substitute Eq. (38) into \(s\dot{s}\) and apply Eq. (39)

Multiply both sides of Eq. (38) by \(s\):

\[ s\dot{s} = s\,d(t) - K\,s\,\tanh\!\left(\frac{s}{\phi}\right) \]

Since \(|s\,d(t)| \leq |s|\Delta\) (disturbance bound) and \(K s\tanh(s/\phi) \geq 0\) (Eq. 39):

Eq. (40) \[ \dot{V}(t) \leq |s|\Delta - K\,s\,\tanh\!\left(\frac{s}{\phi}\right) - \tilde{\lambda}\,s^2 - \tilde{T}_p\,s^2 - \tilde{\phi}\,|s| \]

8.5  The Stability Condition — Eq. (41)

Looking at Eq. (40): the first term \(|s|\Delta\) is destabilising (it wants to increase \(V\)), while the second term \(K s\tanh(s/\phi)\) is stabilising (it decreases \(V\)). The other three terms also decrease \(V\). The controller wins whenever the stabilising term outweighs the disturbance:

Stability condition — Equation (41) \[ K > \Delta \]

This single inequality is the only design requirement for stability. Set the switching gain \(K\) larger than the worst-case disturbance bound \(\Delta\), and the PSSMC is guaranteed stable — regardless of initial conditions, parameter drifts, or the specific realisation of the disturbance, as long as it stays bounded.

🏋️

A Physical Intuition for \(K > \Delta\)

Imagine two people in a tug-of-war. \(\Delta\) is the disturbance pulling the system away from \(s = 0\); \(K\) is the controller pulling it back. As long as the controller is stronger (\(K > \Delta\)), it always wins the tug-of-war. The PSSMC uses \(K = 500\) — chosen to exceed the largest expected chatter force in the Kasprowiak operating envelope.

The Full Proof at a Glance

Step A — Define \(V(t)\) [Eq. 33]

Sum of squared surface value and squared parameter errors. A sum of squares is always \(\geq 0\). Zero only at the equilibrium.

Step B — Differentiate \(V\) [Eq. 35]

Straightforward chain rule. Each squared term yields a product of the variable and its rate of change.

Step C — Substitute adaptive laws [Eq. 36–37]

The \(-\alpha_i s^2\) and \(-\alpha_3|s|\) terms from Page 3 enter. With \(\gamma_i = \alpha_i\), the weights cancel and three dissipative terms remain.

Step D — Substitute closed-loop \(\dot{s}\) [Eq. 38]

The PSSMC control law enters through \(\dot{s} = d(t) - K\tanh(s/\phi)\). The disturbance \(d(t)\) is bounded by \(\Delta\).

Step E — Apply tanh inequality [Eq. 39]

\(s\tanh(s/\phi) \geq 0\) always. This lets us replace \(-Ks\tanh(s/\phi)\) with \(-K|s|\) in the bound, giving a clean expression.

Step F — Invoke stability condition \(K > \Delta\) [Eq. 41]

When \(K > \Delta\), the stabilising term \(K|s|\) dominates the disturbance term \(|s|\Delta\) for \(|s|\) sufficiently large. \(\dot{V} < 0\) outside the boundary layer. QED. ∎

Why do we need Lyapunov?

Simulation can only test a finite number of scenarios. No matter how many runs you do, there could always be an untested initial condition or disturbance profile that causes the controller to fail.

Lyapunov's method, developed by Aleksandr Lyapunov in 1892, provides a way to prove stability for all possible scenarios simultaneously — without simulating each one. The idea: find a function \(V(t) \geq 0\) (like total energy) that is always decreasing. If such a function exists, the system must converge — there is simply no mathematical way for it to diverge.

For the PSSMC, this certificate means the controller is safe to deploy in a real CNC machine — not just likely to work, but provably guaranteed to work within the stated conditions.

The energy analogy — why it works

Physical intuition: if you can show the total energy of a system is always decreasing, the system must eventually come to rest. A pendulum with friction always stops — because every swing dissipates energy.

\(V(t)\) in Lyapunov analysis plays the role of "energy". It is not necessarily real physical energy — it is a mathematical construct chosen to make the proof work. The choice of \(V\) is the art; proving \(\dot{V} \leq 0\) is the craft.

For the PSSMC, \(V = \tfrac{1}{2}s^2 + \text{parameter errors}\) captures both the chatter energy (via \(s^2\)) and the parameter adaptation quality. When \(\dot{V} \leq 0\), both are simultaneously decreasing — chatter is suppressed and gains are converging.

Practical vs asymptotic stability

Asymptotic stability means \(s(t) \to 0\) exactly as \(t \to \infty\). It requires \(\dot{V} < 0\) everywhere except the origin. This would need \(\phi \to 0\), which reintroduces chattering.

Practical stability (also called ultimate boundedness) means \(s(t)\) enters and stays within a small neighbourhood \(|s| \leq \epsilon\) for some \(\epsilon > 0\). Inside this neighbourhood, \(\dot{V}\) may occasionally be slightly positive, but the state cannot escape.

For engineering purposes, practical stability is sufficient — we do not need the vibration to reach exactly zero, only to stay below the threshold at which chatter causes surface damage. The adaptive \(\phi\) law automatically finds the smallest \(\phi\) (tightest neighbourhood) that avoids chattering at each operating point.

The tanh inequality — Eq. (39)

\[ s\,\tanh(s/\phi) \geq 0 \]

This looks simple but it is the keystone of the proof. Here is why it is true:

Case \(s > 0\): \(\tanh(s/\phi) > 0\) (tanh of a positive number is positive). Product is positive.

Case \(s < 0\): \(\tanh(s/\phi) < 0\) (tanh of a negative number is negative). Negative × negative = positive.

Case \(s = 0\): both sides are zero. Product is zero.

In all cases, \(s\) and \(\tanh(s/\phi)\) have the same sign, so their product \(\geq 0\). This allows us to replace \(-K s\tanh(s/\phi)\) with \(-K|s|\) in the bound, which is what makes the algebra tractable.

Understanding the \(\dot{V}\) Graph — Plain Language

The graph has three coloured lines. Each line tells one part of the stability story. Together, they show — visually — why the PSSMC is mathematically guaranteed to suppress chatter. Here is every line explained, from left to right.

The horizontal axis — \(s(t)\)

The x-axis represents the value of the sliding surface \(s(t)\). When \(s = 0\), the machine is perfectly on the surface — chatter is suppressed, force error is decaying exponentially, and the controller has done its job. Moving right along the axis means the machine is drifting further and further away from the desired behaviour: chatter is getting worse, the force error is growing. So the entire story of stability is the story of what pulls \(s\) back toward zero.

The teal line — \(K s \tanh(s / \phi)\) — the controller pushing back

This is the stabilising force — the part of the controller that actively fights to bring \(s\) back to zero.

Notice its shape: it starts at zero when \(s = 0\), rises steeply, and then flattens out as \(s\) grows large. This flattening is the \(\tanh\) function at work — it saturates. The physical meaning: when the machine is very far from the surface, the controller applies its maximum possible restoring force \(K\). It does not keep growing — it caps out. This is what prevents the aggressive, high-frequency switching (chattering) of classical SMC.

The steepness of the teal curve is controlled by \(K\). A larger \(K\) means a steeper rise and a higher plateau. When you drag the \(K\) slider to the right, you are making this curve taller — giving the controller a stronger arm in the tug-of-war.

The red dashed line — \(|s| \Delta\) — the disturbance pushing away

This is the destabilising force — chatter, vibration, modelling errors, anything that tries to push the machine away from \(s = 0\).

It is a straight line through the origin with slope \(\Delta\). This is because we do not know the exact shape of the disturbance — we only know it is bounded by \(|d(t)| \leq \Delta\). The worst case is when every bit of disturbance is pushing in exactly the wrong direction, which gives the linear \(|s|\Delta\) term.

When you drag the \(\Delta\) slider to the right, you are increasing the slope of this line — simulating a more severe cutting environment with stronger chatter forces. A steeper red line means the disturbance is a tougher opponent.

The green line — net \(\dot{V}\) upper bound — the final verdict

This is the green line is the most important one. It shows the net result: disturbance minus controller, i.e. \(|s|\Delta - K s \tanh(s/\phi)\).

If the green line is below zero — the controller is winning. \(\dot{V} < 0\) means the total "energy" \(V(t)\) is decreasing. The system is rolling downhill toward \(s = 0\). Stability is guaranteed.

If the green line is above zero — the disturbance is winning. \(V(t)\) is growing, \(s\) is moving away from zero. The controller is not strong enough. You need to increase \(K\) or the stability condition \(K > \Delta\) is violated.

Notice what happens near \(s = 0\): the green line may briefly be slightly positive, because \(\tanh\) is gentle near zero (it has not yet saturated). This is the boundary layer — a small neighbourhood around \(s = 0\) where we accept that \(\dot{V}\) is not strictly negative. Outside this neighbourhood, the green line drops below zero and stays there. This is practical stability: the system is guaranteed to enter the boundary layer and stay inside it forever.

The crossover point — where control takes over

There is a specific value of \(|s|\) at which the teal curve overtakes the red dashed line. To the right of this crossover, the teal line is higher than the red — the controller is stronger than the disturbance, and the green line is below zero. The system is being pulled back toward \(s = 0\).

This crossover point moves depending on \(K\) and \(\Delta\): increasing \(K\) moves it closer to zero (faster recovery); increasing \(\Delta\) moves it further from zero (slower recovery). When \(K < \Delta\), the crossover never happens — the red line is always above the teal, the green line stays positive, and the system diverges. This is why \(K > \Delta\) is the only design requirement.

Try it on the sliders

  • Set \(K = 200\) and \(\Delta = 400\): the red line dominates — the green line stays positive — unstable. ✗
  • Set \(K = 500\) and \(\Delta = 100\): the teal line rises sharply above the red — the green line drops well below zero — strongly stable. ✓
  • Set \(K = 110\) and \(\Delta = 100\): barely stable — the green line dips below zero only for large \(s\). The boundary layer is wide.
  • Increase \(|s|\) on the first slider to see which term currently dominates at that particular operating point.

In one sentence: the graph shows that as long as \(K > \Delta\), the teal curve always rises above the red dashed line for large \(|s|\), making the green line negative outside the boundary layer — and that is the complete, visual proof that the PSSMC must always suppress chatter.
